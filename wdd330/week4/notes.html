Forms

const form = document.forms['search']; // to get a form called 'search'

Forms have a reset method, but these are considered poor for usability.

Forms have many events associated with them such as change. The change event triggers when an element loses focus after being changed. A submit event occurs when a form is submitted. We can capture this submit with JavaScript.

                        
const form = document.forms['search'];
form.addEventListener ('submit', search, false);

function search() {

Object-Oriented Programming

ES6 introudced classes to JavaScript. These work much like constructor functions, but look much more like classes in other langauges.

                        
// a class in ES6
class Dice {
    constructor(sides=6) {
        this.sides = sides;
    }

    roll() {
        return Math.floor(this.sides * Math.random() + 1)
    }
}

A class prototype is an object so new properties and methods can be assigned to it. All objects made of the class will then have the methods and properties added to the prototype.

className.constructor.prototype; // get a prototype

Use the prototype to add to a class after it has been declared.

Inheritance

You can use "extends" for inheritance. This is more like the traditional inheritance I am used to.

                        
class NinjaTurtle extends Turtle {

        this.weapon = 'hands';
    }
    attack() { return `Feel the power of my ${this.weapon}!` } 
}

Modules

Use modules to seperate code into different files. Use export to export things from modules. Not everything needs to be exported. Use import to import something from a module.

                        
function square(x) {
    return x * x;
}

function sum(array, callback) {
    if(callback) {
        array = array.map(callback);
    }
    return array.reduce((a,b) => a + b );
}

function variance(array) {
    return sum(array,square)/array.length - square(mean(array))
}

function mean(array) {
    return sum(array) / array.length;
}

export {
    variance,
    mean
}
