Further Functions

The call method lets us set the "this" in a function. If the function has any parameters, they must be provided after the first parameter, which is for "this".

                        
function sayHello(){
    return `Hello, my name is ${ this.name }`;
}

const clark = { name: 'Clark' };
const bruce = { name: 'Bruce' };

sayHello.call(clark);
<< 'Hello, my name is Clarke'

sayHello.call(bruce);
<< 'Hello, my name is Bruce'

// using call without "this"
square.call(null, 4)

Functions also have a cache where they can store values. This is helpful if it takes a lot of computational power to get a result from a function.

square.cache

If you want to code in strict mode, but you are not sure if all the code was written in strict mode, then you can write all your code in IIFE.

                        
(function() {
    'use strict';

// All your code would go inside this function

})();

Functions can define, and redefine themselves.

                        
function party() {
    console.log('Wow this is amazing!');
    party = function(){
        console.log('Been there, got the T-Shirt');
    }
}

Ajax

Ajax stands for Asynchronous Javascript and XML. The Fetch API is the current standard for Ajax requests. It uses promises and has a response object.

                        
// fetch with response
fetch(url)
.then( response => response.json() ); // transforms the JSON data into a JavaScript object
.then( data => console.log(Object.entries(data)) )
.catch( error => console.log('There was an error: ', error))

We can use Ajax to asynchronously get data and update our webpage without reloading.
